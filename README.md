reglas del repositorio

 No trabajar directamente en main

Cada par de bounded context tiene su propia branch (rama), trabajen en la branch (rama) que les corresponda

Los cambios llegan a main mediante Pull Request, trabajan en el su branch, hacen sus commits, le dan a pull request (para subir sus cambios al repositorio en la web de la branch) y luego merge para actualizar/añadir sus cambios al main.

Antes de comenzar a trabajar en sus ramas (branchs) recuerden actualizar la branch desde main para tener todo actualizado

No modificar archivos de otros contextos (sin avisar)
