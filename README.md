reglas del repositorio

No trabajar directamente en main

Cada par de bounded context tiene su propia branch (rama), trabajen en la branch (rama) que les corresponda

Los cambios llegan a main mediante Pull Request, trabajan en su branch correspondiente a sus BC, hacen sus commits, le hacen push origin y luego le dan a pull request (para subir sus cambios al repositorio en la web de la branch) y luego merge en el github web para actualizar/añadir sus cambios al main.

Antes de comenzar a trabajar en sus ramas (branchs) recuerden actualizar la branch desde main ("update from main") para tener su rama con los cambios de la versión más actualizada del main.

No modificar archivos de otros contextos (sin avisar)
